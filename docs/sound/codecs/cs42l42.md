## SUMMARY

The CS42L42 is one chip presented to Linux through one shared ASoC core and two thin transport binds. The core in [`cs42l42.c`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c) owns the [`struct cs42l42_private`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27), the [`cs42l42_regmap`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L380) config, the component [`cs42l42_soc_component`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L589), the DAI ops [`cs42l42_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1125), and the whole jack and button detection state machine. The bus layer is thin. [`cs42l42_i2c_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-i2c.c#L16) builds the regmap with [`devm_regmap_init_i2c()`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L1055) and takes the interrupt straight from [`i2c_client->irq`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/i2c.h); [`cs42l42_sdw_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L516) builds a custom regmap whose [`reg_read`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L408) and [`reg_write`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L408) are [`cs42l42_sdw_read()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L221) and [`cs42l42_sdw_write()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L271), and takes the interrupt from a GPIO resource with [`acpi_dev_gpio_irq_get()`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/acpi.h#L1324). Both call [`cs42l42_common_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2278), so the component, the controls, and the jack logic are identical regardless of how the chip is wired.

The DAI differs by transport because the clock and data path differ. The I2C DAI [`cs42l42_dai`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1134) carries [`cs42l42_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1125), whose [`hw_params`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op [`cs42l42_pcm_hw_params()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L908) programs the ASP I2S frame, derives the bit clock, and runs [`cs42l42_pll_config()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L664), and whose [`set_sysclk`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op [`cs42l42_set_sysclk()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L991) records the externally-supplied SCLK. The SoundWire DAI [`cs42l42_sdw_dai`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L177) carries [`cs42l42_sdw_dai_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L167) instead, whose [`hw_params`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op [`cs42l42_sdw_dai_hw_params()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L63) joins the codec to a SoundWire stream with [`sdw_stream_add_slave()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/soundwire/stream.c#L2117) and whose [`prepare`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op [`cs42l42_sdw_dai_prepare()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L98) runs the PLL only once the SoundWire SWIRE_CLK is known from the [`bus_config`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616) callback. The mute op [`cs42l42_mute_stream()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1026) is shared by both DAIs and switches the clock source between the PLL and the internal oscillator.

The two transports also differ in how the device attaches and how registers are reached. On I2C the bus is always present, so [`cs42l42_i2c_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-i2c.c#L16) calls [`cs42l42_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2374) directly. On SoundWire the chip is held in reset until the bus reports it absent, then released; the [`struct sdw_slave_ops`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616) [`cs42l42_sdw_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L406) drive the attach handshake through [`cs42l42_sdw_update_status()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L340), which calls [`cs42l42_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2374) on first attach, and the codec's register I/O is funnelled through the delayed-read protocol in [`cs42l42_sdw_read()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L221). The SoundWire ops carry no [`interrupt_callback`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616), because jack detection on the CS42L42 is reported only on its hardware INT pin, leaving the SoundWire in-band interrupt unused for this purpose.

Jack detection runs entirely in the codec's own threaded IRQ and ends at [`snd_soc_jack_report()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-jack.c#L33). The machine layer registers a [`struct snd_soc_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-jack.h#L80) through [`snd_soc_component_set_jack()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-component.c#L190), which dispatches to [`cs42l42_set_jack()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L563); that op stores the jack in the [`jack`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) field and reports the already-known plug type. A plug edge, a type-detect completion, and a button edge all arrive on the one INT line and are decoded in [`cs42l42_irq_thread()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1662), which reads the sticky status registers, runs [`cs42l42_process_hs_type_detect()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1260) to classify the plug, runs [`cs42l42_handle_button_press()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1535) to map the DC-detect bias level to a button, and reports the matching [`SND_JACK_HEADSET`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/jack.h#L41) or [`SND_JACK_BTN_0`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/jack.h#L50)-family bit.

## SPECIFICATIONS

The CS42L42 on a SoundWire link is a MIPI SoundWire peripheral whose manufacturer id, part id, and class come from the enumeration `_ADR` integer the firmware supplies, matched against the one-entry table [`cs42l42_sdw_id`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L601) built with [`SDW_SLAVE_ENTRY`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L722). The device's two audio data ports follow the SoundWire data-port model the codec joins from its [`hw_params`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op, and the slave attach state machine and bus configuration are the SoundWire bus mechanisms the [`struct sdw_slave_ops`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616) plug into. On I2C the same chip is a plain register peripheral whose 8-bit registers are paged through the [`cs42l42_page_range`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L368) window. The headset accessory-detect logic (tip sense, automatic and manual headset-type detect, and the DC-resistance button decode) is Cirrus device hardware addressed through the [`struct regmap`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L35) register space, and the jack and button reporting model is the Linux software construct [`struct snd_soc_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-jack.h#L80) with [`enum snd_jack_types`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/jack.h#L38), a software description distinct from any hardware register layout. The membership-gated MIPI documents are not linked; this page describes the device from the kernel source and its register names.

## INTERFACES

The CS42L42 is one [`struct cs42l42_private`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) core plus one ASoC component and one DAI, and the only structural difference between the two transports is which regmap, which DAI, and which interrupt source feed the same core. The core, its regmap, its component, and its DAI are all created at bus probe and live for the duration of the bus device bind; the SoundWire path additionally enables runtime PM and arms the slave ops the bus calls back into.

### cs42l42_common_probe and cs42l42_init are the shared entry points

Both transports allocate the same core and reach the same two functions. [`cs42l42_common_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2278) takes the component driver and the DAI driver as parameters, so each bind supplies its own pair (the I2C bind passes [`cs42l42_soc_component`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L589) and [`cs42l42_dai`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1134); the SoundWire bind passes a copy of the component and [`cs42l42_sdw_dai`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L177)). [`cs42l42_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2374) checks the device id, powers the chip, reads device properties, and unmasks interrupts; it runs synchronously on I2C but is deferred to the first SoundWire attach, since registers cannot be reached until the peripheral is on the bus.

### cs42l42_ops and cs42l42_sdw_dai_ops

The two DAI function pointer structs differ because the clock and data path differ. [`cs42l42_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1125) carries the I2S-specific [`set_fmt`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269), [`set_sysclk`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269), and [`set_bclk_ratio`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) for the ASP clock tree. [`cs42l42_sdw_dai_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L167) carries [`set_stream`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) and a [`prepare`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op instead, because a SoundWire DAI joins a stream rather than driving an external clock, and because its PLL cannot be programmed until the bus clock is known. Both ops share [`mute_stream`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) (op [`cs42l42_mute_stream()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1026)), which is the same on both transports.

### set_jack

[`set_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-component.h#L67) is the [`struct snd_soc_component_driver`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-component.h#L67) field the machine driver reaches through [`snd_soc_component_set_jack()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-component.c#L190). The CS42L42 op [`cs42l42_set_jack()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L563) stores the [`struct snd_soc_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-jack.h#L80) in the [`jack`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) field of the core under [`irq_lock`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) to avoid racing the interrupt thread, and immediately reports the current [`hs_type`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) so a jack already plugged in before the machine driver attached is reflected at once. Unlike the per-property accessory-detect programming on some Cirrus codecs, the CS42L42 reads its `cirrus,*` properties earlier, in [`cs42l42_handle_device_data()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1944) during [`cs42l42_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2374).

### The shared core struct ties the codec together

A single [`struct cs42l42_private`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) holds everything the codec shares between the two transports. ### The shared regmap is paged; SoundWire edits a copy

The I2C transport uses the shared [`cs42l42_regmap`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L380) verbatim. Its registers are 8-bit and reached through the paging window [`cs42l42_page_range`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L368), which writes the page number to a selector register and maps a 256-byte window:

```c
/* sound/soc/codecs/cs42l42.c:368 */
const struct regmap_range_cfg cs42l42_page_range = {
	.name = "Pages",
	.range_min = 0,
	.range_max = CS42L42_MAX_REGISTER,
	.selector_reg = CS42L42_PAGE_REGISTER,
	.selector_mask = 0xff,
	.selector_shift = 0,
	.window_start = 0,
	.window_len = 256,
};

const struct regmap_config cs42l42_regmap = {
	.reg_bits = 8,
	.val_bits = 8,

	.readable_reg = cs42l42_readable_register,
	.volatile_reg = cs42l42_volatile_register,

	.ranges = &cs42l42_page_range,
	.num_ranges = 1,

	.max_register = CS42L42_MAX_REGISTER,
	.reg_defaults = cs42l42_reg_defaults,
	.num_reg_defaults = ARRAY_SIZE(cs42l42_reg_defaults),
	.cache_type = REGCACHE_MAPLE,

	.use_single_read = true,
	.use_single_write = true,
};
```

The SoundWire transport cannot use that config directly, because over SoundWire the register address is 16 bits (offset by `0x8000`) and the access is a delayed-read protocol rather than a plain bus read. [`cs42l42_sdw_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L516) copies the shared config with [`devm_kmemdup()`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/string.h) and overrides the register width and the I/O callbacks before building the map:

```c
/* sound/soc/codecs/cs42l42-sdw.c:539 */
	regmap_conf = devm_kmemdup(dev, &cs42l42_regmap, sizeof(cs42l42_regmap), GFP_KERNEL);
	if (!regmap_conf)
		return -ENOMEM;
	regmap_conf->reg_bits = 16;
	regmap_conf->num_ranges = 0;
	regmap_conf->reg_read = cs42l42_sdw_read;
	regmap_conf->reg_write = cs42l42_sdw_write;

	regmap = devm_regmap_init(dev, NULL, peripheral, regmap_conf);
	if (IS_ERR(regmap))
		return dev_err_probe(dev, PTR_ERR(regmap), "Failed to allocate register map\n");

	/* Start in cache-only until device is enumerated */
	regcache_cache_only(regmap, true);
```

The [`reg_read`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L408) callback [`cs42l42_sdw_read()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L221) issues the read, then checks a status byte to learn whether the result was immediate or delayed, polling [`CS42L42_SDW_MEM_READ_DATA`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L32) when the device defers it. From the perspective of the shared core, both maps look identical, so every register access in [`cs42l42.c`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c) works unchanged on either transport.

```
    cs42l42_regmap: an 8-bit register file reached through a page window
    ───────────────────────────────────────────────────────────────────

    write page number to                window maps the selected
    CS42L42_PAGE_REGISTER  ───────────▶  256-byte page into view
    (selector_reg, mask 0xff)            (window_start 0, window_len 256)
                                                   │
                                                   ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │ register file   range 0 .. CS42L42_MAX_REGISTER                 │
    ├──────────────┬──────────────┬──────────────┬────────────────────┤
    │ page 0       │ page 1       │ page 2       │ ...                │
    │ regs 0..255  │ regs 0..255  │ regs 0..255  │                    │
    └──────────────┴──────────────┴──────────────┴────────────────────┘
       reg_bits = 8, val_bits = 8, cache_type = REGCACHE_MAPLE

    SoundWire reuses this config with overrides. reg_bits becomes 16,
    num_ranges 0, and reg_read/reg_write become cs42l42_sdw_read/_write
    (16-bit address + 0x8000, delayed-read protocol). The shared core
    sees one regmap and every cs42l42.c access is transport-agnostic.
```

### The I2C bind is the simplest path

On I2C the bus is always present, so the bind does the whole job up front. [`cs42l42_i2c_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-i2c.c#L16) allocates the core, builds the regmap with [`devm_regmap_init_i2c()`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L1055), takes the interrupt straight off the [`struct i2c_client`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/i2c.h), and calls the shared probe and then [`cs42l42_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2374) directly:

```c
/* sound/soc/codecs/cs42l42-i2c.c:16 */
static int cs42l42_i2c_probe(struct i2c_client *i2c_client)
{
	struct device *dev = &i2c_client->dev;
	struct cs42l42_private *cs42l42;
	struct regmap *regmap;
	int ret;

	cs42l42 = devm_kzalloc(dev, sizeof(*cs42l42), GFP_KERNEL);
	if (!cs42l42)
		return -ENOMEM;

	regmap = devm_regmap_init_i2c(i2c_client, &cs42l42_regmap);
	if (IS_ERR(regmap))
		return dev_err_probe(&i2c_client->dev, PTR_ERR(regmap),
				     "regmap_init() failed\n");

	cs42l42->devid = CS42L42_CHIP_ID;
	cs42l42->dev = dev;
	cs42l42->regmap = regmap;
	cs42l42->irq = i2c_client->irq;

	ret = cs42l42_common_probe(cs42l42, &cs42l42_soc_component, &cs42l42_dai);
	if (ret)
		return ret;

	return cs42l42_init(cs42l42);
}
```

The driver matches the ACPI id `10134242` (the eight-character ACPI hardware id for the part) and registers through [`module_i2c_driver()`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/i2c.h):


### The SoundWire bind defers init to attach

On SoundWire the chip may not be reachable when [`cs42l42_sdw_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L516) runs, so the bind does more setup and leaves [`cs42l42_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2374) for the attach handshake. The probe gets the interrupt from a GPIO resource ([`acpi_dev_gpio_irq_get()`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/acpi.h#L1324) on x86 ACPI), builds the custom regmap shown above, copies the component driver so it can attach the SoundWire-specific DAPM routes, enables runtime PM with a held reference, and calls the shared probe:

```c
/* sound/soc/codecs/cs42l42-sdw.c:554 */
	component_drv = devm_kmemdup(dev,
				     &cs42l42_soc_component,
				     sizeof(cs42l42_soc_component),
				     GFP_KERNEL);
	if (!component_drv)
		return -ENOMEM;

	component_drv->dapm_routes = cs42l42_sdw_audio_map;
	component_drv->num_dapm_routes = ARRAY_SIZE(cs42l42_sdw_audio_map);

	cs42l42->dev = dev;
	cs42l42->regmap = regmap;
	cs42l42->sdw_peripheral = peripheral;
	cs42l42->irq = irq;
	cs42l42->devid = CS42L42_CHIP_ID;
	...
	pm_runtime_get_noresume(cs42l42->dev);
	pm_runtime_enable(cs42l42->dev);

	ret = cs42l42_common_probe(cs42l42, component_drv, &cs42l42_sdw_dai);
	if (ret < 0)
		return ret;

	return 0;
```

The SoundWire path overrides the routes because its data path differs from the ASP. The shared component's [`cs42l42_audio_map`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L532) wires the ASP serial-data widgets, while [`cs42l42_sdw_audio_map`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L41) wires the SRC widgets to the SoundWire `Playback` and `Capture` stream endpoints:

### cs42l42_common_probe powers the chip and registers the component

[`cs42l42_common_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2278) is the transport-agnostic probe both binds reach. It records the core as device drvdata so every callback can recover it, acquires the regulators and the reset GPIO, requests the threaded IRQ on the hardware INT line, and registers the component and DAI it was handed:

```c
/* sound/soc/codecs/cs42l42.c:2278 */
int cs42l42_common_probe(struct cs42l42_private *cs42l42,
			 const struct snd_soc_component_driver *component_drv,
			 struct snd_soc_dai_driver *dai)
{
	int ret, i;

	dev_set_drvdata(cs42l42->dev, cs42l42);
	mutex_init(&cs42l42->irq_lock);
	...
	if (cs42l42->reset_gpio) {
		...
		/*
		 * On SoundWire keep the chip in reset until we get an UNATTACH
		 * notification from the SoundWire core. This acts as a
		 * synchronization point to reject stale ATTACH notifications
		 * if the chip was already enumerated before we reset it.
		 */
		if (cs42l42->sdw_peripheral)
			cs42l42->sdw_waiting_first_unattach = true;
		else
			gpiod_set_value_cansleep(cs42l42->reset_gpio, 1);
	}
	usleep_range(CS42L42_BOOT_TIME_US, CS42L42_BOOT_TIME_US * 2);

	/* Request IRQ if one was specified */
	if (cs42l42->irq) {
		ret = request_threaded_irq(cs42l42->irq,

					   IRQF_ONESHOT | IRQF_TRIGGER_LOW,
					   "cs42l42", cs42l42);
		if (ret) {
			dev_err_probe(cs42l42->dev, ret,
				"Failed to request IRQ\n");
			goto err_disable_noirq;
		}
	}

	/* Register codec now so it can EPROBE_DEFER */
	ret = devm_snd_soc_register_component(cs42l42->dev, component_drv, dai, 1);
	if (ret < 0)
		goto err;

	return 0;
	...
}
```

The reset handling branches on [`sdw_peripheral`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27). According to the comment, on SoundWire the chip is held in reset until the bus reports it absent, so a stale attach from a chip enumerated before the driver reset it can be rejected. On I2C reset is released immediately. The interrupt is requested the same way on both transports, one threaded handler [`cs42l42_irq_thread()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1662) on a level-low oneshot line, because the CS42L42 raises jack-detect events on its own INT pin in both wirings.

### cs42l42_init checks the device id and reads properties

[`cs42l42_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2374) brings the chip up once it is reachable. It validates the device id with [`cirrus_read_device_id()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cirrus_legacy.h#L9), powers up the codec blocks, reads the firmware-supplied detection properties through [`cs42l42_handle_device_data()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1944), applies a SoundWire-only SRC power override, sets up headset detection, and unmasks interrupts:

```c
/* sound/soc/codecs/cs42l42.c:2374 */

{
	unsigned int reg;
	int devid, ret;

	/* initialize codec */
	devid = cirrus_read_device_id(cs42l42->regmap, CS42L42_DEVID_AB);

	 * SRC power is linked to ASP power so doesn't work in Soundwire mode.
	 * Override it and use DAPM to control SRC power for Soundwire.

		regmap_update_bits(cs42l42->regmap, CS42L42_PWR_CTL2,
				   CS42L42_SRC_PDN_OVERRIDE_MASK |
				   CS42L42_DAC_SRC_PDNB_MASK |
				   CS42L42_ADC_SRC_PDNB_MASK,
				   CS42L42_SRC_PDN_OVERRIDE_MASK);

	/* Setup headset detection */
	cs42l42_setup_hs_type_detect(cs42l42);

	 * Set init_done before unmasking interrupts so any triggered

	/* Mask/Unmask Interrupts */
	cs42l42_set_interrupt_masks(cs42l42);

	return 0;
	...
```

According to the comment, [`init_done`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) is set before interrupts are unmasked so an event arriving immediately is handled rather than dropped, and the IRQ thread checks that same flag before touching the chip. On the I2C path [`cs42l42_i2c_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-i2c.c#L16) calls this directly; on SoundWire it is reached from the attach callback described next.

### The SoundWire slave ops drive the attach handshake

The SoundWire bind supplies [`cs42l42_sdw_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L406), the callback surface the bus invokes for property reads, attach transitions, bus configuration, and port preparation. There is deliberately no [`interrupt_callback`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616):

```c
/* sound/soc/codecs/cs42l42-sdw.c:406 */
static const struct sdw_slave_ops cs42l42_sdw_ops = {
/* No interrupt callback because only hardware INT is supported for Jack Detect in the CS42L42 */
	.read_prop = cs42l42_sdw_read_prop,
	.update_status = cs42l42_sdw_update_status,
	.bus_config = cs42l42_sdw_bus_config,
	.port_prep = cs42l42_sdw_port_prep,
};
```

According to the comment, jack detection is reported only on the hardware INT pin, so the codec never registers for the SoundWire in-band interrupt; it relies on the same threaded IRQ the I2C variant uses. The attach handshake is in [`cs42l42_sdw_update_status()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L340), which gates init behind a first-unattach barrier and releases reset when the bus confirms the chip is absent:

```c
/* sound/soc/codecs/cs42l42-sdw.c:340 */
static int cs42l42_sdw_update_status(struct sdw_slave *peripheral,
				     enum sdw_slave_status status)
{
	struct cs42l42_private *cs42l42 = dev_get_drvdata(&peripheral->dev);

	switch (status) {
	case SDW_SLAVE_ATTACHED:
		dev_dbg(cs42l42->dev, "ATTACHED\n");

		/*
		 * The SoundWire core can report stale ATTACH notifications
		 * if we hard-reset CS42L42 in probe() but it had already been
		 * enumerated. Reject the ATTACH if we haven't yet seen an
		 * UNATTACH report for the device being in reset.
		 */
		if (cs42l42->sdw_waiting_first_unattach)
			break;

		/*
		 * Initialise codec, this only needs to be done once.
		 * When resuming from suspend, resume callback will handle re-init of codec,
		 * using regcache_sync().
		 */
		if (!cs42l42->init_done)
			cs42l42_sdw_init(peripheral);
		break;
	case SDW_SLAVE_UNATTACHED:
		dev_dbg(cs42l42->dev, "UNATTACHED\n");

		if (cs42l42->sdw_waiting_first_unattach) {
			/*
			 * SoundWire core has seen that CS42L42 is not on
			 * the bus so release RESET and wait for ATTACH.
			 */
			cs42l42->sdw_waiting_first_unattach = false;
			gpiod_set_value_cansleep(cs42l42->reset_gpio, 1);
		}

		break;
	default:
		break;
	}

	return 0;
}
```

The first [`SDW_SLAVE_UNATTACHED`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L95) clears [`sdw_waiting_first_unattach`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) and releases reset, so the chip re-enumerates cleanly; the following [`SDW_SLAVE_ATTACHED`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L96) then runs [`cs42l42_sdw_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L284). That wrapper takes the regmap out of cache-only, runs the shared [`cs42l42_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2374), syncs any cached register writes that happened before attach, and drops the runtime-PM reference the probe held:

```c
/* sound/soc/codecs/cs42l42-sdw.c:284 */
static void cs42l42_sdw_init(struct sdw_slave *peripheral)
{
	struct cs42l42_private *cs42l42 = dev_get_drvdata(&peripheral->dev);
	int ret;

	regcache_cache_only(cs42l42->regmap, false);

	ret = cs42l42_init(cs42l42);
	if (ret < 0) {
		regcache_cache_only(cs42l42->regmap, true);
		goto err;
	}

	/* Write out any cached changes that happened between probe and attach */
	ret = regcache_sync(cs42l42->regmap);
	if (ret < 0)
		dev_warn(cs42l42->dev, "Failed to sync cache: %d\n", ret);

	/* Disable internal logic that makes clock-stop conditional */
	regmap_clear_bits(cs42l42->regmap, CS42L42_PWR_CTL3, CS42L42_SW_CLK_STP_STAT_SEL_MASK);

err:
	/* This cancels the pm_runtime_get_noresume() call from cs42l42_sdw_probe(). */
	pm_runtime_put_autosuspend(cs42l42->dev);
}
```

The bus drives the status callback through these states in order, the held-in-reset entry rejecting a stale ATTACH until an UNATTACHED releases reset and the next ATTACHED reaches the init wrapper above:

```
    SoundWire attach handshake (cs42l42_sdw_update_status)
    ──────────────────────────────────────────────────────
    Reset is held in probe so a stale ATTACH is rejected until
    the bus first confirms the chip absent.

    ┌──────────────────────────────────────────────────────┐
    │ HELD IN RESET  sdw_waiting_first_unattach = true     │
    │   a stale ATTACH here ─▶ break (stay in reset)       │
    └───────────────────────┬──────────────────────────────┘
                            │ UNATTACHED: bus confirms absent,
                            │ clear wait flag, release reset
                            ▼
    ┌──────────────────────────────────────────────────────┐
    │ RELEASED, waiting for ATTACHED                       │
    └───────────────────────┬──────────────────────────────┘
                            │ ATTACHED and init_done == false
                            ▼
    ┌──────────────────────────────────────────────────────┐
    │ cs42l42_sdw_init ─▶ cs42l42_init, regcache_sync,     │
    │ drop the probe-time PM reference                     │
    └──────────────────────────────────────────────────────┘
```

### The SoundWire hw_params joins a stream, then prepare runs the PLL

The SoundWire DAI splits the work across two ops because its clock is not known until the bus tells it. [`cs42l42_sdw_dai_hw_params()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L63) records the sample rate, fills the stream and port config with [`snd_sdw_params_to_config()`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/sdw.h#L32), picks the playback or capture data port, and joins the codec to the stream with [`sdw_stream_add_slave()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/soundwire/stream.c#L2117):

```c
/* sound/soc/codecs/cs42l42-sdw.c:63 */
static int cs42l42_sdw_dai_hw_params(struct snd_pcm_substream *substream,
				     struct snd_pcm_hw_params *params,
				     struct snd_soc_dai *dai)
{
	struct cs42l42_private *cs42l42 = snd_soc_component_get_drvdata(dai->component);
	struct sdw_stream_runtime *sdw_stream = snd_soc_dai_get_dma_data(dai, substream);
	struct sdw_stream_config stream_config = {0};
	struct sdw_port_config port_config = {0};
	int ret;

	if (!sdw_stream)
		return -EINVAL;

	/* Needed for PLL configuration when we are notified of new bus config */
	cs42l42->sample_rate = params_rate(params);

	snd_sdw_params_to_config(substream, params, &stream_config, &port_config);

	if (substream->stream == SNDRV_PCM_STREAM_PLAYBACK)
		port_config.num = CS42L42_SDW_PLAYBACK_PORT;
	else
		port_config.num = CS42L42_SDW_CAPTURE_PORT;

	ret = sdw_stream_add_slave(cs42l42->sdw_peripheral, &stream_config, &port_config, 1,
				   sdw_stream);
	if (ret) {
		dev_err(dai->dev, "Failed to add sdw stream: %d\n", ret);
		return ret;
	}

	cs42l42_src_config(dai->component, params_rate(params));

	return 0;
}
```

The [`sdw_stream`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L63) handle it reads back was stored by the [`set_stream`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op [`cs42l42_sdw_dai_set_sdw_stream()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L153), a thin wrapper over [`snd_soc_dai_dma_data_set()`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L506). The PLL is not programmed here, because the SoundWire bus clock arrives separately. [`cs42l42_sdw_bus_config()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L386) records the SWIRE_CLK into [`sclk`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) when the bus tells the peripheral, and [`cs42l42_sdw_dai_prepare()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L98) runs the PLL once both halves are known:

```c
/* sound/soc/codecs/cs42l42-sdw.c:98 */
static int cs42l42_sdw_dai_prepare(struct snd_pcm_substream *substream,
				   struct snd_soc_dai *dai)
{
	struct cs42l42_private *cs42l42 = snd_soc_component_get_drvdata(dai->component);

	dev_dbg(dai->dev, "dai_prepare: sclk=%u rate=%u\n", cs42l42->sclk, cs42l42->sample_rate);

	if (!cs42l42->sclk || !cs42l42->sample_rate)
		return -EINVAL;

	/*
	 * At this point we know the sample rate from hw_params, and the SWIRE_CLK from bus_config()
	 * callback. This could only fail if the ACPI or machine driver are misconfigured to allow
	 * an unsupported SWIRE_CLK and sample_rate combination.
	 */

	return cs42l42_pll_config(dai->component, cs42l42->sclk, cs42l42->sample_rate);
}
```

The [`hw_free`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op [`cs42l42_sdw_dai_hw_free()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42-sdw.c#L117) reverses the join with [`sdw_stream_remove_slave()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/soundwire/stream.c#L2228) and clears [`sample_rate`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27).

```
    SoundWire PLL needs two inputs from two callbacks
    ─────────────────────────────────────────────────
    The PLL needs both the bus clock and the sample rate, so the
    prepare callback programs it after both inputs are known.

    ┌────────────────────────────┐   ┌────────────────────────────┐
    │ bus_config callback        │   │ hw_params (DAI op)         │
    │ cs42l42_sdw_bus_config     │   │ cs42l42_sdw_dai_hw_params  │
    │   records SWIRE_CLK        │   │   records params_rate,     │
    │   ─▶ sclk                  │   │   sdw_stream_add_slave     │
    │                            │   │   ─▶ sample_rate           │
    └──────────────┬─────────────┘   └─────────────┬──────────────┘
                   │ sclk                          │ sample_rate
                   └───────────────┬───────────────┘
                                   ▼
                  ┌──────────────────────────────────┐
                  │ prepare: cs42l42_sdw_dai_prepare │
                  │   needs sclk AND sample_rate set │
                  │   ─▶ cs42l42_pll_config(sclk,    │
                  │        sample_rate)              │
                  └──────────────────────────────────┘
```

### mute_stream switches the clock source and is shared

### The jack hook stores the jack and reports the current type

[`cs42l42_set_jack()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L563) is the [`set_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-component.h#L67) op. It stores the machine driver's jack in [`jack`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) under [`irq_lock`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) to avoid racing the interrupt thread, then reports whatever plug type detection has already found:

```c
/* sound/soc/codecs/cs42l42.c:563 */
static int cs42l42_set_jack(struct snd_soc_component *component, struct snd_soc_jack *jk, void *d)
{
	struct cs42l42_private *cs42l42 = snd_soc_component_get_drvdata(component);

	/* Prevent race with interrupt handler */
	mutex_lock(&cs42l42->irq_lock);
	cs42l42->jack = jk;

	if (jk) {
		switch (cs42l42->hs_type) {
		case CS42L42_PLUG_CTIA:
		case CS42L42_PLUG_OMTP:
			snd_soc_jack_report(jk, SND_JACK_HEADSET, SND_JACK_HEADSET);
			break;
		case CS42L42_PLUG_HEADPHONE:
			snd_soc_jack_report(jk, SND_JACK_HEADPHONE, SND_JACK_HEADPHONE);
			break;
		default:
			break;
		}
	}
	mutex_unlock(&cs42l42->irq_lock);

	return 0;
}
```

### One hardware interrupt decodes plug, type, and button

The CS42L42 raises one INT line, and [`cs42l42_irq_thread()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1662) decodes everything from it. It runs in process context where it may sleep and call into DAPM through the report path. It reads the twelve sticky status banks named by [`irq_params_table`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1635), masking each with its enable register, then dispatches type detect, tip sense, and button detect:

```c
/* sound/soc/codecs/cs42l42.c:1662 */
```

The thread first takes the [`irq_lock`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) the [`set_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-component.h#L67) op also holds, so the [`jack`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) pointer cannot change underneath it, and bails out early when the chip is suspended or not yet initialized. The type-detect branch fires when the auto-detect-done bit is set; it classifies the plug and reports the headset or headphone bit against the full headset-plus-buttons mask, so a new plug clears any stale button bits:

```c
/* sound/soc/codecs/cs42l42.c:1705 */
```

The tip-sense branch tracks the plug state. A transition to plugged starts a fresh hardware type-detect cycle with [`cs42l42_init_hs_type_detect()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1399); a transition to unplugged cancels detection and clears the whole jack with the wide mask:

```c
/* sound/soc/codecs/cs42l42.c:1730 */
	if ((~masks[11]) & irq_params_table[11].mask) {
		switch (current_plug_status) {
		case CS42L42_TS_PLUG:

				cs42l42->plug_state = CS42L42_TS_PLUG;

		case CS42L42_TS_UNPLUG:
			if (cs42l42->plug_state != CS42L42_TS_UNPLUG) {
				cs42l42->plug_state = CS42L42_TS_UNPLUG;
				cs42l42_cancel_hs_type_detect(cs42l42);

				snd_soc_jack_report(cs42l42->jack, 0,
						    SND_JACK_HEADSET |
						    SND_JACK_BTN_0 | SND_JACK_BTN_1 |
						    SND_JACK_BTN_2 | SND_JACK_BTN_3);

				dev_dbg(cs42l42->dev, "Unplug event\n");
			}
			break;

		default:
			cs42l42->plug_state = CS42L42_TS_TRANS;
```

The button branch runs only while a headset is plugged. A falling edge (button down) reads the resistance and reports the matching button; a rising edge (release) clears the button bits:

```c
/* sound/soc/codecs/cs42l42.c:1759 */
```

### Button detection sweeps the DC-detect bias levels

[`cs42l42_handle_button_press()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1535) classifies which button was pressed by sweeping the level-detect sensitivity through the four [`bias_thresholds`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.h#L27) entries until the headset-true bit stops asserting, then maps the level it stopped at to a [`SND_JACK_BTN_*`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/jack.h#L50) bit:

```c
/* sound/soc/codecs/cs42l42.c:1557 */
	/* Test all 4 level detect biases */
	bias_level = 1;
	do {
		/* Adjust button detect level sensitivity */
		regmap_update_bits(cs42l42->regmap,
			CS42L42_MIC_DET_CTL1,
			CS42L42_LATCH_TO_VP_MASK |
			CS42L42_EVENT_STAT_SEL_MASK |
			CS42L42_HS_DET_LEVEL_MASK,
			(1 << CS42L42_LATCH_TO_VP_SHIFT) |
			(0 << CS42L42_EVENT_STAT_SEL_SHIFT) |
			(cs42l42->bias_thresholds[bias_level] <<
			CS42L42_HS_DET_LEVEL_SHIFT));

		regmap_read(cs42l42->regmap, CS42L42_DET_STATUS2,
				&detect_status);
	} while ((detect_status & CS42L42_HS_TRUE_MASK) &&
		(++bias_level < CS42L42_NUM_BIASES));

	switch (bias_level) {
	case 1: /* Function C button press */
		bias_level = SND_JACK_BTN_2;
		dev_dbg(cs42l42->dev, "Function C button press\n");
		break;
	case 2: /* Function B button press */
		bias_level = SND_JACK_BTN_1;
		dev_dbg(cs42l42->dev, "Function B button press\n");
		break;
	case 3: /* Function D button press */
		bias_level = SND_JACK_BTN_3;
		dev_dbg(cs42l42->dev, "Function D button press\n");
		break;
	case 4: /* Function A button press */
		bias_level = SND_JACK_BTN_0;
		dev_dbg(cs42l42->dev, "Function A button press\n");
		break;
	default:
		bias_level = 0;
		break;
	}
```

The four levels map to the four standard headset functions, so the returned bit is the [`SND_JACK_BTN_0`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/jack.h#L50) through [`SND_JACK_BTN_3`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/jack.h#L53) value the IRQ thread passes straight to [`snd_soc_jack_report()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-jack.c#L33). The thresholds it compares against were programmed from device properties earlier; [`cs42l42_handle_device_data()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1944) reads the `cirrus,bias-lvls` array (and the tip-sense and button debounce properties) during [`cs42l42_init()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L2374), falling back to [`threshold_defaults`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1937) when the firmware supplies none.

```
    cs42l42_handle_button_press: bias level maps to a button bit
    ────────────────────────────────────────────────────────────
    Sweep bias_thresholds[bias_level] for level = 1..4 until the
    HS_TRUE bit stops asserting; the level it stopped at decides:

    ┌─────────────┬──────────────┬─────────────────────────────┐
    │ bias_level  │ HS function  │ reported bit                │
    ├─────────────┼──────────────┼─────────────────────────────┤
    │ 1           │ Function C   │ SND_JACK_BTN_2              │
    │ 2           │ Function B   │ SND_JACK_BTN_1              │
    │ 3           │ Function D   │ SND_JACK_BTN_3              │
    │ 4           │ Function A   │ SND_JACK_BTN_0              │
    │ none        │ (no match)   │ 0                           │
    └─────────────┴──────────────┴─────────────────────────────┘

    thresholds come from cirrus,bias-lvls (or threshold_defaults),
    read once by cs42l42_handle_device_data during cs42l42_init.
```

### snd_soc_jack_report is the single endpoint

Every CS42L42 detection path, plug, type, and button, ends at [`snd_soc_jack_report()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-jack.c#L33). It updates the masked bits of [`status`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-jack.h#L80) under the jack mutex, walks the pin list to enable or disable each DAPM pin bound to the jack, runs the notifier chain, syncs DAPM, and reports the new state to the input layer through [`snd_jack_report()`](https://elixir.bootlin.com/linux/v7.0/source/sound/core/jack.c#L629):

```c
/* sound/soc/soc-jack.c:33 */
void snd_soc_jack_report(struct snd_soc_jack *jack, int status, int mask)
{
	struct snd_soc_dapm_context *dapm;
	struct snd_soc_jack_pin *pin;
	unsigned int sync = 0;

	if (!jack || !jack->jack)
		return;
	trace_snd_soc_jack_report(jack, mask, status);

	dapm = snd_soc_card_to_dapm(jack->card);

	mutex_lock(&jack->mutex);

	jack->status &= ~mask;
	jack->status |= status & mask;

	trace_snd_soc_jack_notify(jack, status);

	list_for_each_entry(pin, &jack->pins, list) {
		int enable = pin->mask & jack->status;

		if (pin->invert)
			enable = !enable;

		if (enable)
			snd_soc_dapm_enable_pin(dapm, pin->pin);
		else
			snd_soc_dapm_disable_pin(dapm, pin->pin);

		/* we need to sync for this case only */
		sync = 1;
	}

	/* Report before the DAPM sync to help users updating micbias status */
	blocking_notifier_call_chain(&jack->notifier, jack->status, jack);

	if (sync)
		snd_soc_dapm_sync(dapm);

	snd_jack_report(jack->jack, jack->status);

	mutex_unlock(&jack->mutex);
}
```

Because the report path takes the jack mutex and synchronizes DAPM, it must run where it can sleep, which is why the CS42L42 reports from its threaded IRQ [`cs42l42_irq_thread()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l42.c#L1662) and not from a hard interrupt context. The masked update lets the codec keep the plug and button bit groups independent. A type-detect report sets the headset or headphone bit against the full mask, a button press sets only its button bit, and an unplug clears the whole jack at once. The same report endpoint and the same threaded IRQ serve both the I2C and the SoundWire wiring, since only the regmap, the DAI, and the interrupt source differ between them.

```
    snd_soc_jack_report: each event sets only its own bit group
    ────────────────────────────────────────────────────────────
    status &= ~mask;  then OR-in (arg & mask)   (per call)

    event (in cs42l42_irq_thread)   status arg       mask arg
    ─────────────────────────────   ──────────────   ───────────────
    type detect: CTIA / OMTP        SND_JACK_HEADSET  HEADSET +
    type detect: 3-pole             SND_JACK_HEADPHONE  BTN_0..3
    button press                    BTN_n (level)    BTN_0..3 only
    button release                  0                BTN_0..3 only
    unplug (TS_UNPLUG)              0                HEADSET + BTN_0..3

    ┌────────────────────────────┐   ┌────────────────────────────┐
    │ plug bit group             │   │ button bit group           │
    │ SND_JACK_HEADSET or        │   │ SND_JACK_BTN_0 .. BTN_3    │
    │ SND_JACK_HEADPHONE         │   │                            │
    └────────────────────────────┘   └────────────────────────────┘
       set by the type-detect path      set by the button branch;
       independent: a button report     a new plug clears the
       never disturbs the plug bit      stale button bits
```
