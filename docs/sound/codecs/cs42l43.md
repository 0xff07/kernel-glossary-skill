## SUMMARY

The CS42L43 is one chip presented to Linux as a [`struct cs42l43`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) MFD core plus three child platform devices. The bus layer is thin. A SoundWire probe [`cs42l43_sdw_probe()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43-sdw.c#L170) or an I2C probe allocates the core, builds its [`struct regmap`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L35) over the transport with [`devm_regmap_init_sdw()`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L1196), and calls the transport-agnostic [`cs42l43_dev_probe()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L1049). That core probe records the core as device drvdata, powers the chip, and queues [`cs42l43_boot_work()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L903), which waits for the device to attach, reads the device id, runs the firmware update, builds the IRQ domain in [`cs42l43_irq_config()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L856), and registers the cells with [`devm_mfd_add_devices()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/core.c#L391).

The IRQ model is the join between the MFD core and the codec. The device raises one hardware interrupt. On SoundWire that line is [`cs42l43->sdw->irq`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L665). [`cs42l43_irq_config()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L856) copies the [`cs42l43_irq_chip`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L496) template into the core's [`irq_chip`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) field and calls [`devm_regmap_add_irq_chip()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/base/regmap/regmap-irq.c#L984), which decodes the device's interrupt status and mask registers into a domain of many named sub-interrupts. The codec cell never touches the SoundWire interrupt path. Its probe [`cs42l43_codec_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L2284) finds that domain through the parent firmware node and requests one threaded IRQ per detection event from the [`cs42l43_irqs`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L2183) table.

The audio cell is an ordinary ASoC codec. [`cs42l43_codec_probe()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L2284) recovers the core through `pdev->dev.parent`, allocates a [`struct cs42l43_codec`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L43), requests the detection IRQs, and registers the component [`cs42l43_component_drv`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L2159) and the DAI array [`cs42l43_dais`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L655) with [`devm_snd_soc_register_component()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-devres.c#L29). The first DAI is the ASP I2S/TDM port using [`cs42l43_asp_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L620), and the next seven are SoundWire data ports DP1 through DP7 sharing [`cs42l43_sdw_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L642), whose [`hw_params`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op [`cs42l43_sdw_hw_params()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L629) joins the codec to a SoundWire stream by reaching the parent [`struct sdw_slave`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L665) and calling [`sdw_stream_add_slave()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/soundwire/stream.c#L2117).

Jack detection runs through the codec's threaded IRQs and ends at [`snd_soc_jack_report()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-jack.c#L33). The machine layer registers a [`struct snd_soc_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-jack.h#L80) through [`snd_soc_component_set_jack()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-component.c#L190), which dispatches to [`cs42l43_set_jack()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L70). That op stores the jack in [`jack_hp`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L43) and programs the accessory-detect registers from device properties. A plug edge fires the tip-sense IRQ [`cs42l43_tip_sense()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L796), which debounces into [`cs42l43_tip_sense_work()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L734); the worker runs [`cs42l43_run_type_detect()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L661) to classify the plug and reports the result. A button edge fires [`cs42l43_button_press()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L368), which reads the DC-detect resistance and reports the matching [`SND_JACK_BTN_0`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/jack.h#L50)-shifted bit.

## SPECIFICATIONS

The CS42L43 on a SoundWire link is a MIPI SoundWire peripheral whose manufacturer id, part id, version, and class id come from the enumeration `_ADR` integer the firmware supplies, split into a [`struct sdw_slave_id`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L480) during attach. The device's audio data ports follow the SoundWire data-port model the codec joins from its [`hw_params`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) op, and the in-band interrupt and slave attach state machine are the SoundWire bus mechanisms the core slave ops plug into. The headset accessory-detect logic (tip and ring sense, type detect, DC-resistance button decode) is Cirrus device hardware addressed through the [`struct regmap`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L35) register space, and the jack and button reporting model is the Linux software construct [`struct snd_soc_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-jack.h#L80) with [`enum snd_jack_types`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/jack.h#L38), a software description distinct from any hardware register layout. The membership-gated MIPI documents are not linked; this page describes the device from the kernel source and its register names.

## INTERFACES

The CS42L43 is one core object plus three child platform devices, and the audio cell builds the ASoC component and the live IRQ mappings on top of the core. The core and its IRQ chip template are created at bus probe and live as long as the device is bound; the cells and the codec's per-event IRQ mappings are created later by the boot work and the codec probe.

### cs42l43_sdw_ops (the MFD slave ops) and cs42l43_sdw_ops (the codec DAI ops)

Two distinct objects share the symbol name [`cs42l43_sdw_ops`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43-sdw.c#L163). In [`drivers/mfd/cs42l43-sdw.c`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43-sdw.c#L163) it is the [`struct sdw_slave_ops`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616) the SoundWire bus calls for attach, interrupt, and bus configuration of the whole device. In [`sound/soc/codecs/cs42l43.c`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L642) it is the [`struct snd_soc_dai_ops`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) the ASoC core calls for the SoundWire data-port DAIs. Each is `static const` and local to its file, so the two never collide at link time.

### set_jack

[`set_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-component.h#L67) is the [`struct snd_soc_component_driver`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-component.h#L67) field the machine driver reaches through [`snd_soc_component_set_jack()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-component.c#L190). The CS42L43 op [`cs42l43_set_jack()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L70) stores the [`struct snd_soc_jack`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-jack.h#L80) in [`jack_hp`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L43) and reads the button-resistance and debounce device properties to program the accessory-detect registers. The same op is invoked from [`cs42l43_component_remove()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L2145) with a NULL jack to clear the configuration at teardown.

### set_stream, hw_params, and hw_free (SoundWire DAIs)

The DP1 through DP7 DAIs share [`cs42l43_sdw_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L642). [`set_stream`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) (op [`cs42l43_sdw_set_stream()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-sdw.c#L61)) stores the SoundWire stream handle with [`snd_soc_dai_dma_data_set()`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L506). [`hw_params`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) (op [`cs42l43_sdw_hw_params()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L629)) reads it back, joins the codec to the stream with [`sdw_stream_add_slave()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/soundwire/stream.c#L2117), and programs the sample rate. [`hw_free`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) (op [`cs42l43_sdw_remove_peripheral()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-sdw.c#L47)) reverses it with [`sdw_stream_remove_slave()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/soundwire/stream.c#L2228). The ASP DAI uses [`cs42l43_asp_ops`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.c#L620) instead, which carries [`set_fmt`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) and [`set_tdm_slot`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269) for the I2S/TDM clock tree and no [`set_stream`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-dai.h#L269).

### update_status and interrupt_callback (MFD slave ops)

The MFD core's [`cs42l43_sdw_ops`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43-sdw.c#L163) carries the bus-facing callbacks. [`update_status`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616) (op [`cs42l43_sdw_update_status()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43-sdw.c#L93)) does not touch jack state; it sets the [`attached`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) flag and completes the [`device_attach`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) or [`device_detach`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) completion the boot work and reset paths wait on. [`interrupt_callback`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616) (op [`cs42l43_sdw_interrupt()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43-sdw.c#L123)) runs after the regmap-irq handler has already dispatched the real interrupt and only clears the Cirrus general-interrupt status register that the SoundWire framework does not cover.

## DETAILS

### The MFD core struct ties the device together

A single [`struct cs42l43`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) holds everything the whole device shares. It carries the [`regmap`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) used for all register access, the [`sdw`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) slave (NULL when the chip is wired over I2C), the single hardware [`irq`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) line, the embedded [`irq_chip`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) and the [`irq_data`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) domain handle, the [`boot_work`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) that brings the chip up, and the completions the attach handshake signals:

```c
/* include/linux/mfd/cs42l43.h:73 */
struct cs42l43 {
	struct device *dev;
	struct regmap *regmap;
	struct sdw_slave *sdw;

	struct regulator *vdd_p;
	struct regulator *vdd_d;
	struct regulator_bulk_data core_supplies[CS42L43_N_SUPPLIES];

	struct gpio_desc *reset;

	int irq;
	struct regmap_irq_chip irq_chip;
	struct regmap_irq_chip_data *irq_data;

	struct work_struct boot_work;
	struct completion device_attach;
	struct completion device_detach;
	struct completion firmware_download;
	int firmware_error;

	unsigned int sdw_freq;
	/* Lock to gate control of the PLL and its sources. */
	struct mutex pll_lock;

	bool sdw_pll_active;
	bool attached;
	bool hw_lock;
};
```

The [`irq_chip`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) field is a full [`struct regmap_irq_chip`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L1707) embedded directly as a struct value, which lets [`cs42l43_irq_config()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L856) copy the static template into it and then stamp the per-instance [`irq_drv_data`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L1707) back pointer.

### The bus probe is thin and transport-agnostic

The SoundWire side does almost nothing of its own. [`cs42l43_sdw_probe()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43-sdw.c#L170) allocates the core, records the [`struct sdw_slave`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L665), builds the regmap over the SoundWire transport, and hands control to the shared core probe:

```c
/* drivers/mfd/cs42l43-sdw.c:170 */
static int cs42l43_sdw_probe(struct sdw_slave *sdw, const struct sdw_device_id *id)
{
	struct cs42l43 *cs42l43;
	struct device *dev = &sdw->dev;

	cs42l43 = devm_kzalloc(dev, sizeof(*cs42l43), GFP_KERNEL);
	if (!cs42l43)
		return -ENOMEM;

	cs42l43->dev = dev;
	cs42l43->sdw = sdw;

	cs42l43->regmap = devm_regmap_init_sdw(sdw, &cs42l43_sdw_regmap);
	if (IS_ERR(cs42l43->regmap))
		return dev_err_probe(cs42l43->dev, PTR_ERR(cs42l43->regmap),
				     "Failed to allocate regmap\n");

	return cs42l43_dev_probe(cs42l43);
}
```

The bus driver supplies the SoundWire slave ops the bus calls back into, [`cs42l43_sdw_ops`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43-sdw.c#L163):

The I2C transport in [`drivers/mfd/cs42l43-i2c.c`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43-i2c.c) reaches the same [`cs42l43_dev_probe()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L1049) with a regmap built over I2C, so the cells and their drivers are identical regardless of how the chip is wired.

### cs42l43_dev_probe powers the chip and queues the boot work

[`cs42l43_dev_probe()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L1049) is the transport-agnostic core probe. It records the core as device drvdata so every child and every callback can recover it, initializes the attach completions and the [`boot_work`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73), acquires the regulators and the reset GPIO, powers the device, and queues the boot work on a long workqueue:

```c
/* drivers/mfd/cs42l43.c:1049 */
int cs42l43_dev_probe(struct cs42l43 *cs42l43)
{
	int i, ret;

	dev_set_drvdata(cs42l43->dev, cs42l43);

	mutex_init(&cs42l43->pll_lock);
	init_completion(&cs42l43->device_attach);
	init_completion(&cs42l43->device_detach);
	init_completion(&cs42l43->firmware_download);
	INIT_WORK(&cs42l43->boot_work, cs42l43_boot_work);

	regcache_cache_only(cs42l43->regmap, true);
	...
	queue_work(system_long_wq, &cs42l43->boot_work);

	return 0;
}
EXPORT_SYMBOL_NS_GPL(cs42l43_dev_probe, "MFD_CS42L43");
```

The probe returns immediately. The slow work of waiting for the device to attach over the bus, reading the device id, and updating firmware happens in the queued work so the bus probe does not block. The device is held active across that window by a `pm_runtime_get_noresume()` that the boot work later releases.

### One hardware interrupt fans out through the regmap IRQ chip

The device raises one interrupt line. The static [`cs42l43_irq_chip`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L496) describes how to decode it into many named sub-interrupts. It names the base status and mask registers and the count of register banks, and points at the descriptor array that assigns each device interrupt its index:

```c
/* drivers/mfd/cs42l43.c:496 */
static const struct regmap_irq_chip cs42l43_irq_chip = {
	.name = "cs42l43",

	.status_base = CS42L43_DECIM_INT,
	.mask_base = CS42L43_DECIM_MASK,
	.num_regs = 16,

	.irqs = cs42l43_regmap_irqs,
	.num_irqs = ARRAY_SIZE(cs42l43_regmap_irqs),

	.runtime_pm = true,
};
```

[`cs42l43_irq_config()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/mfd/cs42l43.c#L856) builds the live domain from that template. On SoundWire it takes the interrupt line from the slave ([`cs42l43->sdw->irq`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L665)), copies the template into the core's [`irq_chip`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) so it can stamp the per-instance [`irq_drv_data`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/regmap.h#L1707), reads the firmware-described trigger type, and registers the chip with [`devm_regmap_add_irq_chip()`](https://elixir.bootlin.com/linux/v7.0/source/drivers/base/regmap/regmap-irq.c#L984):

```c
/* drivers/mfd/cs42l43.c:856 */
static int cs42l43_irq_config(struct cs42l43 *cs42l43)
{
	struct irq_data *irq_data;
	unsigned long irq_flags;
	int ret;

	if (cs42l43->sdw)
		cs42l43->irq = cs42l43->sdw->irq;

	cs42l43->irq_chip = cs42l43_irq_chip;
	cs42l43->irq_chip.irq_drv_data = cs42l43;

	irq_data = irq_get_irq_data(cs42l43->irq);
	if (!irq_data) {
		dev_err(cs42l43->dev, "Invalid IRQ: %d\n", cs42l43->irq);
		return -EINVAL;
	}

	irq_flags = irqd_get_trigger_type(irq_data);
	switch (irq_flags) {
	case IRQF_TRIGGER_LOW:
	case IRQF_TRIGGER_HIGH:
	case IRQF_TRIGGER_RISING:
	case IRQF_TRIGGER_FALLING:
		break;
	case IRQ_TYPE_NONE:
	default:
		irq_flags = IRQF_TRIGGER_LOW;
		break;
	}

	irq_flags |= IRQF_ONESHOT;

	ret = devm_regmap_add_irq_chip(cs42l43->dev, cs42l43->regmap,
				       cs42l43->irq, irq_flags, 0,
				       &cs42l43->irq_chip, &cs42l43->irq_data);
	if (ret) {
		dev_err(cs42l43->dev, "Failed to add IRQ chip: %d\n", ret);
		return ret;
	}
	...
	return 0;
}
```

After this runs, the device's interrupt is delivered to the regmap-irq handler, which reads the 16 status banks, finds the asserted device interrupts, and dispatches each as a separate Linux IRQ on the domain in [`cs42l43->irq_data`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73). The codec cell consumes those IRQs. The SoundWire [`interrupt_callback`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616) runs only as cleanup, since the real interrupt was already handled by the regmap-irq path:

```c
/* drivers/mfd/cs42l43-sdw.c:123 */
static int cs42l43_sdw_interrupt(struct sdw_slave *sdw,
				 struct sdw_slave_intr_status *status)
{
	/*
	 * The IRQ itself was handled through the regmap_irq handler, this is
	 * just clearing up the additional Cirrus SoundWire registers that are
	 * not covered by the SoundWire framework or the IRQ handler itself.
	 * There is only a single bit in GEN_INT_STAT_1 and it doesn't clear if
	 * IRQs are still pending so doing a read/write here after handling the
	 * IRQ is fine.
	 */
	sdw_read_no_pm(sdw, CS42L43_GEN_INT_STAT_1);
	sdw_write_no_pm(sdw, CS42L43_GEN_INT_STAT_1, CS42L43_INT_STAT_GEN1_MASK);

	return 0;
}
```

The attach handshake the core probe waits on is signalled from the [`update_status`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/soundwire/sdw.h#L616) op. It unmasks the general interrupt, flips the [`attached`](https://elixir.bootlin.com/linux/v7.0/source/include/linux/mfd/cs42l43.h#L73) flag, and completes the matching completion:

```c
/* drivers/mfd/cs42l43-sdw.c:93 */
static int cs42l43_sdw_update_status(struct sdw_slave *sdw, enum sdw_slave_status status)
{
	struct cs42l43 *cs42l43 = dev_get_drvdata(&sdw->dev);

	switch (status) {
	case SDW_SLAVE_ATTACHED:
		dev_dbg(cs42l43->dev, "Device attach\n");

		sdw_write_no_pm(sdw, CS42L43_GEN_INT_MASK_1,
				CS42L43_INT_STAT_GEN1_MASK);

		cs42l43->attached = true;

		complete(&cs42l43->device_attach);
		break;
	case SDW_SLAVE_UNATTACHED:
		dev_dbg(cs42l43->dev, "Device detach\n");

		cs42l43->attached = false;

		reinit_completion(&cs42l43->device_attach);
		complete(&cs42l43->device_detach);
		break;
	default:
		break;
	}

	return 0;
}
```

Once attached, that one interrupt line feeds the regmap IRQ chip, whose handler reads the 16 status banks and splits each asserted bit into its own Linux IRQ on the domain the codec's handlers request:

```
    One hardware IRQ line fans out into named device interrupts
    ───────────────────────────────────────────────────────────

    cs42l43->irq  (one line; sdw->irq on SoundWire)
              │  devm_regmap_add_irq_chip(&cs42l43->irq_chip)
              ▼
    ┌──────────────────────────────────────────────────────┐
    │ struct regmap_irq_chip  cs42l43_irq_chip             │
    │   status_base = CS42L43_DECIM_INT                    │
    │   mask_base   = CS42L43_DECIM_MASK                   │
    │   num_regs    = 16   (status / mask register banks)  │
    │   irqs        = cs42l43_regmap_irqs[]                │
    └────────────────────────────┬─────────────────────────┘
                                 │ handler reads 16 banks,
                                 │ one Linux IRQ per asserted bit
                                 ▼  (domain in cs42l43->irq_data)
    ┌──────────────────────────────────────────────────────┐
    │ cs42l43_irqs[]  codec maps + requests each (threaded)│
    │                                                      │
    │   CS42L43_HSDET_DONE         ▶ cs42l43_type_detect   │
    │   CS42L43_TIPSENSE_*_PDET    ▶ cs42l43_tip_sense     │
    │   CS42L43_DC_DETECT1_TRUE    ▶ cs42l43_button_press  │
    │   CS42L43_DC_DETECT1_FALSE   ▶ cs42l43_button_release│
    └──────────────────────────────────────────────────────┘
```

### The plug path debounces into a worker, then classifies

A plug edge fires the tip-sense IRQ. [`cs42l43_tip_sense()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L796) does no classification itself; it cancels any pending work and re-queues [`tip_sense_work`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L43) on the debounce delay so a noisy edge settles, extending the delay after a resume to avoid a false detection:

```c
/* sound/soc/codecs/cs42l43-jack.c:796 */
irqreturn_t cs42l43_tip_sense(int irq, void *data)
{
	struct cs42l43_codec *priv = data;
	unsigned int db_delay = priv->tip_debounce_ms;

	cancel_delayed_work(&priv->bias_sense_timeout);
	cancel_delayed_work(&priv->tip_sense_work);

	// Ensure delay after suspend is long enough to avoid false detection
	if (priv->suspend_jack_debounce)
		db_delay += priv->tip_fall_db_ms + priv->tip_rise_db_ms;

	queue_delayed_work(system_long_wq, &priv->tip_sense_work,
			   msecs_to_jiffies(db_delay));

	return IRQ_HANDLED;
}
```

[`cs42l43_tip_sense_work()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L734) runs in process context where it may sleep and call into DAPM. It reads the tip and ring sense status, and on a present jack runs [`cs42l43_run_type_detect()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L661) to classify the plug, then reports the resulting mask into [`jack_hp`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L43). On removal it clears the jack with the wide `0xFFFF` mask:

```c
/* sound/soc/codecs/cs42l43-jack.c:734 */

{
	struct cs42l43_codec *priv = container_of(work, struct cs42l43_codec,
						  tip_sense_work.work);
	struct cs42l43 *cs42l43 = priv->core;
	unsigned int sts = 0;

	int ret, report;
	...
	regmap_read(cs42l43->regmap, CS42L43_TIP_RING_SENSE_INTERRUPT_STATUS, &sts);

	tip = (sts >> CS42L43_TIPSENSE_PLUG_DB_STS_SHIFT) & CS42L43_JACK_PRESENT;
	ring = (sts >> CS42L43_RINGSENSE_PLUG_DB_STS_SHIFT) & CS42L43_JACK_PRESENT;

	if (tip == CS42L43_JACK_PRESENT) {
		...
		if (priv->use_ring_sense && ring == CS42L43_JACK_ABSENT) {
			report = CS42L43_JACK_OPTICAL;
		} else {
			report = cs42l43_run_type_detect(priv);
			if (report < 0) {
				dev_err(priv->dev, "Jack detect failed: %d\n", report);
				goto error;
			}
		}

		snd_soc_jack_report(priv->jack_hp, report, report);
	} else {
		priv->jack_override = 0;

		cs42l43_clear_jack(priv);

		snd_soc_jack_report(priv->jack_hp, 0, 0xFFFF);
		...
	}
	...
}
```

The two report forms differ by mask. On a present jack the worker reports `report` against `report` so it sets exactly the bits the type detect found; on removal it reports 0 against `0xFFFF` to clear everything at once. [`cs42l43_run_type_detect()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L661) drives the hardware type detect, waits on a completion that the type-detect IRQ fires, reads the headset-status register, and maps the type field to either a load detect or an open-circuit extension:

```c
/* sound/soc/codecs/cs42l43-jack.c:661 */

{
	struct cs42l43 *cs42l43 = priv->core;
	int timeout_ms = ((2 * priv->detect_us) / USEC_PER_MSEC) + 200;
	unsigned int type = 0xff;
	unsigned long time_left;

	reinit_completion(&priv->type_detect);
	...
	cs42l43_start_hs_bias(priv, true);
	regmap_update_bits(cs42l43->regmap, CS42L43_HS2,
			   CS42L43_HSDET_MODE_MASK, 0x3 << CS42L43_HSDET_MODE_SHIFT);

	time_left = wait_for_completion_timeout(&priv->type_detect,
						msecs_to_jiffies(timeout_ms));
	...
	if (!time_left)
		return -ETIMEDOUT;

	regmap_read(cs42l43->regmap, CS42L43_HS_STAT, &type);
	...
	switch (type & CS42L43_HSDET_TYPE_STS_MASK) {
	case 0x0: // CTIA
	case 0x1: // OMTP
	case 0x4:
		return cs42l43_run_load_detect(priv, true);
	case 0x2: // 3-pole
		return cs42l43_run_load_detect(priv, false);
	case 0x3: // Open-circuit
		return CS42L43_JACK_EXTENSION;
	default:
		return -EINVAL;
	}
}
```

A CTIA or OMTP four-pole plug runs [`cs42l43_run_load_detect()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L618) with the microphone path, which returns [`CS42L43_JACK_HEADSET`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L29) and arms button detection; a three-pole plug runs load detect without the microphone and returns headphone or line-out by the measured impedance. The resulting [`CS42L43_JACK_*`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L29) value is the report mask the worker passes to [`snd_soc_jack_report()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-jack.c#L33).

### The button path runs entirely in its threaded IRQ

The button press is handled directly in the threaded IRQ, with no intermediate work item. [`cs42l43_button_press()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L368) waits for the comb filter to settle, reads the DC-detect status, converts it to a resistance, walks the [`buttons`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L43) threshold array to find the matching button index, and reports the corresponding [`SND_JACK_BTN_0`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/jack.h#L50) bit shifted by that index under the [`CS42L43_JACK_BUTTONS`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L29) mask:

```c
/* sound/soc/codecs/cs42l43-jack.c:368 */
irqreturn_t cs42l43_button_press(int irq, void *data)
{
	struct cs42l43_codec *priv = data;
	struct cs42l43 *cs42l43 = priv->core;
	irqreturn_t iret = IRQ_NONE;
	unsigned int buttons = 0;
	unsigned int val = 0;
	int i, ret;

	ret = pm_runtime_resume_and_get(priv->dev);
	if (ret) {
		dev_err(priv->dev, "Failed to resume for button press: %d\n", ret);
		return iret;
	}

	mutex_lock(&priv->jack_lock);

	if (!priv->button_detect_running) {
		dev_dbg(priv->dev, "Spurious button press IRQ\n");
		goto error;
	}

	// Wait for 2 full cycles of comb filter to ensure good reading
	usleep_range(2 * CS42L43_BUTTON_COMB_US, 2 * CS42L43_BUTTON_COMB_US + 50);

	regmap_read(cs42l43->regmap, CS42L43_DETECT_STATUS_1, &val);

	/* Bail if jack removed, the button is irrelevant and likely invalid */
	if (!cs42l43_jack_present(priv)) {
		dev_dbg(priv->dev, "Button ignored due to removal\n");
		goto error;
	}
	...
	val = (val & CS42L43_HSDET_DC_STS_MASK) >> CS42L43_HSDET_DC_STS_SHIFT;
	val = ((CS42L43_BUTTON_COMB_MAX << 20) / (val + 1)) - (1 << 20);
	if (val)
		val = (CS42L43_BUTTON_ROUT << 20) / val;
	else
		val = UINT_MAX;

	for (i = 0; i < CS42L43_N_BUTTONS; i++) {
		if (val < priv->buttons[i]) {
			buttons = SND_JACK_BTN_0 >> i;
			dev_dbg(priv->dev, "Detected button %d at %d Ohms\n", i, val);
			break;
		}
	}

	if (!buttons)
		dev_dbg(priv->dev, "Unrecognised button: %d Ohms\n", val);

	snd_soc_jack_report(priv->jack_hp, buttons, CS42L43_JACK_BUTTONS);

	iret = IRQ_HANDLED;

error:
	mutex_unlock(&priv->jack_lock);

	pm_runtime_put_autosuspend(priv->dev);

	return iret;
}
```

The release is a separate IRQ. [`cs42l43_button_release()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L436) reports 0 under the same [`CS42L43_JACK_BUTTONS`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L29) mask, clearing the button bits while leaving the plug bits the type-detect path set untouched:

### snd_soc_jack_report is the single endpoint

Every CS42L43 detection path, whether the type-detect worker or the button IRQs, ends at [`snd_soc_jack_report()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/soc-jack.c#L33). It updates the masked bits of [`status`](https://elixir.bootlin.com/linux/v7.0/source/include/sound/soc-jack.h#L80) under the jack mutex, walks the pin list to enable or disable each DAPM pin bound to the jack, runs the notifier chain so bias consumers can react, syncs DAPM, and reports the new state to the input layer through [`snd_jack_report()`](https://elixir.bootlin.com/linux/v7.0/source/sound/core/jack.c#L470):

```c
/* sound/soc/soc-jack.c:33 */
void snd_soc_jack_report(struct snd_soc_jack *jack, int status, int mask)
{
	struct snd_soc_dapm_context *dapm;
	struct snd_soc_jack_pin *pin;
	unsigned int sync = 0;

	if (!jack || !jack->jack)
		return;
	trace_snd_soc_jack_report(jack, mask, status);

	dapm = snd_soc_card_to_dapm(jack->card);

	mutex_lock(&jack->mutex);

	jack->status &= ~mask;
	jack->status |= status & mask;

	trace_snd_soc_jack_notify(jack, status);

	list_for_each_entry(pin, &jack->pins, list) {
		int enable = pin->mask & jack->status;

		if (pin->invert)
			enable = !enable;

		if (enable)
			snd_soc_dapm_enable_pin(dapm, pin->pin);
		else
			snd_soc_dapm_disable_pin(dapm, pin->pin);

		/* we need to sync for this case only */
		sync = 1;
	}

	/* Report before the DAPM sync to help users updating micbias status */
	blocking_notifier_call_chain(&jack->notifier, jack->status, jack);

	if (sync)
		snd_soc_dapm_sync(dapm);

	snd_jack_report(jack->jack, jack->status);

	mutex_unlock(&jack->mutex);
}
```

Because the report path takes the jack mutex and synchronizes DAPM, it must run where it can sleep, which is why the CS42L43 plug detection reports from [`cs42l43_tip_sense_work()`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43-jack.c#L734) and the button detection reports from threaded IRQs rather than from any hard interrupt context. The masked update lets the codec keep the plug and button bit groups independent. A type-detect report sets the headset, headphone, or line-out bits without disturbing buttons, a button press sets only its [`CS42L43_JACK_BUTTONS`](https://elixir.bootlin.com/linux/v7.0/source/sound/soc/codecs/cs42l43.h#L29) bits without disturbing the plug state, and a removal clears the whole jack with the wide mask.

```
    snd_soc_jack_report: one masked update keeps bit groups apart
    ────────────────────────────────────────────────────────────
    status &= ~mask;  then OR-in (arg & mask)        (per call)

    event source            status arg    mask arg
    ──────────────────────  ───────────   ──────────────────────
    type detect (present)   report        report  (only its bits)
    button press            BTN_n bit     CS42L43_JACK_BUTTONS
    button release          0             CS42L43_JACK_BUTTONS
    jack removal            0             0xFFFF  (clears all)

    ┌────────────────────────────┐   ┌────────────────────────────┐
    │ plug bits                  │   │ button bits                │
    │ HEADSET / headphone /      │   │ CS42L43_JACK_BUTTONS       │
    │ line-out / EXTENSION       │   │                            │
    └────────────────────────────┘   └────────────────────────────┘
       set by the type-detect path      set by the button IRQs;
       a button report leaves these     a plug report leaves
       untouched                        these untouched
```
